# SKILLS_GUIDE.md — 技能库编写指南

> 本文档说明如何向 `backend/data/skills.json` 添加新技能，以及技能如何被 AI 系统调用。

---

## 一、文件位置

```
game_chat/
└── backend/
    └── data/
        └── skills.json   ← 技能库文件
```

---

## 二、skills.json 数据结构

文件为一个 JSON 数组，每项为一个技能对象：

```json
[
  {
    "id": "cold_open",
    "category": "破冰",
    "name": "冷启动开场",
    "desc": "对方对你兴趣不大时，用反直觉的开头打破预期。",
    "usage": "初次搭讪 / 对话冷场后重新启动",
    "example": "你不像本地人。",
    "tags": ["破冰", "吸引力", "悬念"],
    "related_styles": ["极简冷感款", "反客为主款"]
  }
]
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一 ID，小写英文+下划线 |
| `category` | string | ✅ | 技能分类（见下方分类表） |
| `name` | string | ✅ | 技能名称，简短有力（4-10字） |
| `desc` | string | ✅ | 技能描述，说明「做什么、为什么有效」（30-100字） |
| `usage` | string | ✅ | 适用场景，逗号分隔 |
| `example` | string | ✅ | 具体示例话术（1-2句，真实可用） |
| `tags` | array | ✅ | 标签列表，用于搜索和分类 |
| `related_styles` | array | ❌ | 关联的回复风格（见五大风格） |

## 三、技能分类（category）

| 分类 | 说明 |
|------|------|
| `破冰` | 陌生情境下的第一句话 |
| `吸引力` | 制造吸引、引发好奇 |
| `推拉` | 给予→抽离的张力制造 |
| `幽默` | 调侃、自嘲、搞笑话术 |
| `深入` | 加深了解、引导袒露 |
| `升温` | 推进关系、增加亲密感 |
| `约会` | 邀约、确认、安排见面 |
| `挽回` | 关系修复、重建联系 |
| `节奏` | 控制回复节奏、不急不慢 |
| `框架` | 建立主导权和叙事框架 |

## 四、五大回复风格

技能的 `related_styles` 字段引用以下风格名称：

| 风格 | 图标 | 核心 |
|------|------|------|
| `极简冷感款` | 🧊 | 少即是多，制造距离感和神秘感 |
| `反客为主款` | ♟️ | 主动掌控话语权，打破对方节奏 |
| `幽默调侃款` | 😏 | 轻松有趣，用笑声拉近距离 |
| `温柔推进款` | 🌊 | 顺势而为，推动关系自然升温 |
| `深度连接款` | 🔗 | 精神共鸣，让对方感到被理解 |

## 五、技能与 game_master.md 的关系

`game_master.md` 是最高优先级的决策框架（CPO框架），定义了：
- **C**ontrol（主导权）
- **P**ush-Pull（推拉张力）
- **O**utcome（目标导向）

技能库中的技能是 CPO 框架的**具体执行工具**。

AI 生成回复时的调用顺序：
```
game_master.md（方向）→ skills.json（技巧）→ 五大风格（表达形式）
```

## 六、编写优质技能的原则

1. **具体可执行** — 示例话术要能直接用，不能写"表现出自信"这种空洞描述
2. **有反直觉性** — 有效技能往往违反直觉（如"不要急着回复"）
3. **场景清晰** — usage 字段要写清楚什么时候用、什么时候不该用
4. **简短有力** — desc 控制在 100 字以内，example 在 2 句以内
5. **避免说教** — 技能文案要像老司机分享经验，不要像心理学教材

## 八、维护建议

- 每次添加技能后，重启后端服务使技能生效（技能在每次分析时动态加载）
- ID 一旦确定不要修改（可能被其他地方引用）
- 定期删除效果差的技能，保持技能库精炼（建议不超过 50 条）
- 可按 category 分文件管理，再合并为 skills.json
